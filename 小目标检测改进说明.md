# YOLO11 小目标检测改进方案

## 项目概述
基于原始的 `YOLO11-HFAMPAN-AsDDet-NWD.yaml` 配置文件，针对小目标检测进行了专门优化，创建了多个版本的配置文件。

## 可用的配置文件

### 1. YOLO11-P2-Simple.yaml ✅ (推荐使用)
**状态**: 测试通过，可正常运行
**特点**: 
- 基于原始配置进行最小修改
- 增加P2层特征图用于小目标检测
- 保持原有HFAMPAN和AsDDet结构
- 使用NWD损失函数
- 三尺度检测：P2/4, P3/8, P4/16

**适用场景**: 
- 需要稳定可靠的小目标检测方案
- 对模型复杂度有限制的场景
- 初次尝试小目标检测优化

### 2. YOLO11-SmallObject-Enhanced.yaml ✅ (功能增强)
**状态**: 基于成功版本扩展，理论可运行
**特点**:
- 在P2-Simple基础上增加MFFF多频率特征融合
- 专门针对P2层进行小目标特征增强
- 保持稳定的基础架构

**适用场景**:
- 需要更强小目标检测能力
- 对精度要求较高的场景

### 3. YOLO11-HFAMPAN-AsDDet-NWD-SmallObject-PRR-v3.yaml ✅ (当前推荐)
**状态**: 已完成接线修复并作为当前默认训练配置
**特点**:
- PRR-v3 三尺度检测架构（P2/P3/P4）
- 已正式接入 AsDDet 检测头
- 已修复 SSDS 尺度监督生效逻辑
- 支持 baseline / ab / full 三种训练模式

## 小目标检测关键改进

### 1. 多尺度特征金字塔
- **P2层 (1/4分辨率)**: 专门检测极小目标 (0-16像素)
- **P3层 (1/8分辨率)**: 检测小目标 (16-32像素)
- **P4层 (1/16分辨率)**: 检测中等目标 (32-96像素)
- **P5层 (1/32分辨率)**: 检测大目标 (96+像素)

### 2. 小目标检测专用模块
- **MFFF (多频率特征融合)**: 利用频域信息增强小目标特征
- **FrequencyFocusedDownSampling**: 频率聚焦下采样，保留高频信息
- **SemanticAlignmenCalibration**: 语义对齐校准，对齐不同尺度特征
- **HFAMPAN**: 分层特征聚合金字塔注意力网络
- **AsDDet**: 自适应空间分解检测头

### 3. 损失函数优化
- **NWD损失函数**: 专门针对小目标检测优化的损失函数

## 使用建议

### 快速开始 (推荐)
```bash
# 使用当前推荐的 PRR-v3 完整链路
python "train_yolo111 copy.py" --trainer_mode full --cfg YOLO11-HFAMPAN-AsDDet-NWD-SmallObject-PRR-v3.yaml --data data_VD.yaml --batch 4 --enable_ssds --debug_routing
```

### 高精度需求
```bash
# 仅验证 PRR-v3 结构
python "train_yolo111 copy.py" --trainer_mode baseline --cfg YOLO11-HFAMPAN-AsDDet-NWD-SmallObject-PRR-v3.yaml --data data_VD.yaml --batch 4
```

## 项目中发现的即插即用小目标模块

### 1. UAV相关模块 (ultralytics/nn/core11/uav.py)
- `DySample_`: 动态上采样模块
- `SPDConv`: 空间金字塔下采样卷积
- `MFFF`: 多频率特征融合模块
- `FrequencyFocusedDownSampling`: 频率聚焦下采样
- `SemanticAlignmenCalibration`: 语义对齐校准模块

### 2. 特征增强模块 (ultralytics/nn/core11/GDM.py)
- `TopBasicLayer`: 顶层基础层
- `PyramidPoolAgg`: 金字塔池化聚合
- `LAF_h`: 特征对齐模块
- `InjectionMultiSum_Auto_pool*`: 特征注入模块
- `RepBlock`: 重参数化块

### 3. 检测头模块 (ultralytics/nn/modules/Head/)
- `AsDDet`: 自适应空间分解检测头

## 注意事项

1. **通道数匹配**: 在修改配置时需要特别注意各层之间的通道数匹配
2. **模块参数**: 不同模块的参数格式需要严格按照源码要求
3. **层索引引用**: YAML文件中的层索引引用需要准确计算
4. **测试验证**: 建议在使用前先运行测试脚本验证配置正确性
